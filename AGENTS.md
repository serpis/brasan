# AGENT HANDBOK

## Projektöversikt
- **Syfte:** En statisk webbplats som skapar en mysig inomhusbrasa med animerad SVG, interaktiva vedträn, tidsreglage och en bränsleplott.
- **Teknikstack:** Ren HTML/CSS/JS – inga byggsteg behövs. Öppna `index.html` i valfri webbläsare.
- **Nyckelfiler**
  - `index.html` – markup för eldstad, kontroller och grafkortet.
  - `styles.css` – hela den visuella upplevelsen (tegel, plattor, flame-scale via CSS-variabler).
  - `script.js` – allt beteende: vedhantering, tidsskalning, bränsledynamik och canvas-plotten.

## Kod & Arkitektur
- **Statehantering:** `script.js` kapslar logiken i `init()` som körs när DOM är redo. Där skapas:
  - `logs` (Map) för individuella vedträn (för- och efterbränning).
  - Bränslemodell (`fuelLevel`) som exponentiellt decayer (`FUEL_DECAY`) och fylls med konstant (`FUEL_PER_LOG`).
  - `fuelHistory` + `drawChart()` för att rita linje/area på canvas (senaste 60 s simulerad tid).
- **Animationer:** CSS använder `--fire-speed`, `--flame-scale-x/y`, `--glow-scale`, `--ember-scale` och `--flame-opacity`; JS uppdaterar dem efter tidsskalning respektive bränslenivå.
- **Kontroller:** Tidshastighet (`#time-speed`) styr både JS-simulering och CSS-animation; knappen `#add-log` lägger till loggar/fuel.
- **Ljud:** Web Audio (`createCrackleEngine`) genererar sprakande bakgrund. `#toggle-audio` startar/stoppar ljudet och kräver användarinteraktion för att aktivera `AudioContext`.

## Arbetsflödesregler (ärvda från tidigare agent)
1. **Shell-kommandon**: använd `["bash","-lc","<cmd>"]` och sätt alltid `workdir`.
2. **Sökning**: föredra `rg`; om saknas använd alternativ.
3. **Filsystem**: `workspace-write` – du kan läsa/skriva inom repo utan extra godkännande.
4. **Planering**: Använd `update_plan` för större/bredare uppgifter (ej triviala). Minst två steg, uppdatera efter varje steg.
5. **Redigering**: Föredra `apply_patch` för manuella ändringar; inga autoformatterare utan instruktion. Håll kommentarer korta.
6. **Externa förändringar**: Rör inte användarens befintliga ändringar om de inte ber om det. Inga destruktiva `git reset` etc.
7. **Rapportering**: Slutsvar ska vara kort, utan att dumpa filer, och följa CLI-formatreglerna (kort text, bulletar vid behov, referera filer med `path:line`).

## Tips till nästa agent
- Förhandsgranska sidan ofta; `open index.html` räcker (ingen bundler behövs).
- Vid ny funktionalitet, kontrollera att eventlyssnare är kopplade (`init()` skippar om DOM-element saknas).
- Skulle du behöva fler resurser (bilder, ljud etc.) nämn det i svaret eftersom vi saknar nätverkstillgång.
- Om grafer eller animationer känns sega, laborera med konstanterna högst upp i `script.js`.
